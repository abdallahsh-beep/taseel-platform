# روزنامة تأصيل — منصة إدارة المحتوى (نموذج أولي)

منصة إدارة المحتوى وسير العمل لجمعية تأصيل التعليمية، مبنية وفق وثيقة المتطلبات `../PRD.md` وبهوية الجمعية البصرية (ألوان وخطوط Qomra وSakkal Saad).

## التشغيل

**الطريقة الأسهل:** انقر نقراً مزدوجاً على ملف «تشغيل المنصة.bat» في المجلد الرئيسي — سيشغّل الخادم ويفتح المتصفح تلقائياً.

أو يدوياً:

```bash
npm install
npx prisma migrate dev   # ينشئ قاعدة البيانات المحلية (SQLite)
node prisma/seed.js      # بيانات تجريبية للربع الثالث 2026
npm run dev              # http://localhost:3000
```

## ما المنفَّذ (من المرحلة الأولى في الـ PRD)

- الروزنامة: شهري + **ربع سنوي بمحاكاة الروزنامة الورقية** + قائمة، بتقويم مزدوج هجري (أم القرى)/ميلادي وطبقة مناسبات، وسحب وإفلات لتغيير المواعيد (FR-01..06, FR-37) + **تصدير PDF/طباعة** (FR-33)
- بطاقات المحتوى: نص أساسي + نسخة لكل منصة مع **عداد أحرف** بحدود كل منصة، و**معاينة المنشور** بشكل المنصة (FR-07..10)
- كانبان بأعمدة الحالات، والانتقالات محكومة بآلة الحالات وبالأدوار — الاعتماد والنشر لا يتمان بالسحب (FR-14..17)
- اعتماد/رفض بملاحظة إلزامية مع سجل اعتمادات، وأي تعديل على محتوى معتمد (نصاً أو مرفقات) يعيده للاعتماد تلقائياً (FR-16/17/25)
- «بانتظاري» للمشرف و«مستحق النشر» لمسؤول النشر مقسمة زمنياً مع **توثيق النشر برابط لكل منصة** (FR-18, FR-34/35)
- **مكتبة الوسائط**: رفع (صور/PDF/فيديو حتى 20MB)، مجلدات ووسوم، **إصدارات لكل أصل**، وربط إصدار محدد بالبطاقة (FR-23..26)
- **الإشعارات داخل المنصة**: جرس بعداد + صفحة إشعارات — تتولد عند الإسناد والذكر وطلب/قرار الاعتماد والنشر (FR-27)
- التعليقات مع @الإشارة + **تعليقات توضيحية (Annotations) على مقاطع النص** (FR-11, FR-40)
- **روابط معاينة الضيوف** لبطاقة أو شهر كامل: قراءة فقط، بانتهاء صلاحية، بلا كشف أسماء أو تعليقات داخلية (FR-39)
- **البحث الشامل** في العناوين والنصوص والهاشتاقات والتعليقات (FR-30)
- **الاستيراد من CSV** لترحيل الروزنامة الحالية من Excel (FR-32)
- لوحة معلومات إحصائية وسجل نشاط كامل (FR-31)

## غير المنفَّذ بعد (حسب أولويات الـ PRD)

الإشعارات بالبريد الإلكتروني (FR-28 — تتطلب SMTP)، إصدارات نصوص المنشورات (FR-13)، المواعيد الداخلية للتسليم (FR-12)، الاعتماد متعدد المستويات (البنية جاهزة).

## ملاحظات تقنية

- **قاعدة البيانات:** Prisma على SQLite للتطوير المحلي؛ للإنتاج غيّر `provider` إلى `postgresql` في `prisma/schema.prisma` — المخطط مطابق للقسم 9 من الـ PRD.
- **الشعار:** المستخدم حالياً شعار نصي مؤقت؛ ضع الشعار الرسمي في `public/brand/logo.png` وسيُستخدم تلقائياً.
- **الخطوط:** ملفات TTF في `src/fonts/` (Qomra للعناوين، Sakkal Saad للنصوص).
- غيّر `SESSION_SECRET` في `.env` قبل أي نشر فعلي.
- **روابط الضيوف تعرض المحتوى المعتمد فقط** (حالتا «جاهز» و«منشور») — المسودات الداخلية لا تُكشف. لتغيير الحالات المرئية للضيوف عدّل `GUEST_VISIBLE_STATUSES` في `src/app/share/[token]/page.tsx`.
- خضع الكود لمراجعة أمنية آلية (رفع الملفات، روابط الضيوف، الصلاحيات) وأُصلحت النتائج المؤكدة: منع تحريك بطاقات الغير، إغلاق ثغرة تحديث متغيرات بطاقات أخرى في توثيق النشر، ضبط حد حجم الطلب لرفع الوسائط، وحصر كشف الضيوف بالمحتوى المعتمد.
